# Whim 多租户与权限架构

## 整体关系

```mermaid
flowchart TD
    U["sys_user<br/>全局用户账号"] --> UT["sys_user_tenant<br/>用户所属租户"]
    T["sys_tenant<br/>租户"] --> UT

    T --> TP["sys_tenant_package<br/>租户套餐"]
    TP --> TPP["sys_tenant_package_permission<br/>套餐可用权限"]
    P["sys_permission<br/>菜单/按钮权限"] --> TPP

    U --> UR["sys_user_role<br/>用户租户角色"]
    T --> UR
    R["sys_role<br/>角色<br/>data_scope"] --> UR

    R --> RP["sys_role_permission<br/>角色权限"]
    T --> RP
    P --> RP

    U --> UP["sys_user_post<br/>用户租户岗位"]
    T --> UP
    Post["sys_post<br/>岗位"] --> UP

    D["sys_dept<br/>部门树"] --> Post

    R --> RD["sys_role_dept<br/>自定义数据部门"]
    D --> RD
    T --> RD
```

## 登录与租户切换

```mermaid
flowchart TD
    Login["用户登录"] --> IsSuper{"是否超级管理员"}

    IsSuper -->|是| Platform["进入平台管理端<br/>currentTenantId 为空"]
    Platform --> SelectTenant["选择租户"]
    SelectTenant --> TenantContext["进入租户上下文<br/>currentTenantId = 选中租户"]

    IsSuper -->|否| CountTenant{"绑定几个租户"}
    CountTenant -->|1 个| DirectTenant["直接进入该租户"]
    CountTenant -->|多个| FirstTenant["默认进入第一个租户<br/>允许右上角切换"]

    DirectTenant --> TenantContext
    FirstTenant --> TenantContext
    TenantContext --> Refresh["刷新当前租户下的<br/>角色/权限/岗位/数据范围"]
```

## 关系说明

`sys_user` 是全局账号，不直接代表某个租户身份。一个用户可以通过 `sys_user_tenant` 进入多个租户。

`currentTenantId` 表示当前正在操作的租户。普通用户只能切换自己绑定的租户，超级管理员可以进入平台管理端，也可以切换到任意租户。

`sys_tenant_package` 和 `sys_tenant_package_permission` 定义租户套餐能够使用的权限上限。

`sys_user_role` 定义用户在当前租户下拥有的角色，`sys_role_permission` 定义角色拥有的菜单和按钮权限。用户最终功能权限应当来自租户套餐权限和当前租户角色权限的交集。

`sys_dept` 是租户下的部门树，`sys_post` 是租户下的岗位，用户通过 `sys_user_post` 获得当前租户下的岗位身份。岗位应当承载组织位置，再由岗位推导用户所在部门。

`sys_role.data_scope` 定义角色的数据范围，`sys_role_dept` 用于自定义部门数据权限。

数据权限最终在当前租户内生效：先按 `currentTenantId` 限制租户边界，再按角色数据范围限制部门、本部门及以下、本人或自定义部门。

## 权限计算

```text
功能权限 = 租户套餐允许的权限 ∩ 当前租户角色授予的权限
数据权限 = 当前租户边界 + 当前租户角色的数据范围
组织归属 = 当前租户 + 用户岗位 + 岗位所属部门
```
